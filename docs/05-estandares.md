# Estándares del proyecto

## Código

- TypeScript estricto.
- No usar `any`.
- No usar lógica duplicada.
- Componentes pequeños y reutilizables.
- Una responsabilidad por componente.

## Arquitectura

- Feature First.
- Barrel Exports.
- Imports absolutos.
- Separación entre UI, lógica y datos.

## Estilos

- Tailwind CSS.
- No usar estilos inline.
- Design Tokens.
- Variables CSS para colores.

## Accesibilidad

- Navegable por teclado.
- aria-label cuando sea necesario.
- Contraste AA como mínimo.

## Performance

- Lazy Loading.
- Dynamic Imports.
- next/image.
- Optimización de fuentes.

## Calidad

- ESLint sin errores.
- TypeScript sin warnings.
- Tests obligatorios para la lógica importante.
